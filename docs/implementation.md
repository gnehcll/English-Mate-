**目标**：项目能跑起来，数据库能连上。

- [ ] 确认 `npm run dev` 正常启动
- [ ] 清理默认首页，替换为空白模板

## 阶段 2：认证与用户系统

**目标**：用户可以注册和登录。

- [ ] 创建 `src/lib/auth.ts`（NextAuth 配置）
- [ ] 创建 API 路由 `src/app/api/auth/[...nextauth]/route.ts`
- [ ] 创建注册页面 `/auth/register`
- [ ] 创建登录页面 `/auth/login`
- [ ] 创建认证中间件（保护需要登录的页面）
- [ ] 导航栏显示登录状态（已登录显示用户菜单，未登录显示登录按钮）

**验证**：注册 → 登录 → 访问受保护页面 → 退出登录 → 无法访问受保护页面。

---

## 阶段 3：设置页面

**目标**：用户可以配置 AI 提供商和 API Key。

- [ ] 创建 `/settings` 页面
- [ ] 提供商选择下拉框（OpenAI / Claude / DeepSeek / 通义千问）
- [ ] API Key 输入框（密码类型，加密存储）
- [ ] 模型选择下拉框（根据提供商动态变化）
- [ ] 创建 API 路由保存/读取设置
- [ ] 创建 `src/lib/encryption.ts`（AES 加密/解密）

**验证**：登录 → 进入设置 → 选择提供商 → 输入 API Key → 选择模型 → 保存 → 刷新页面仍然存在。

---

## 阶段 4：写作与翻译页面

**目标**：核心内容输入页面可用。

- [ ] 创建 `/write` 页面
- [ ] Tab 切换：自由写作 / 中译英
- [ ] 自由写作模式：

  - 英文文章编辑器
- [ ] 中译英模式：
  - 中文原文输入框
  - 英文翻译输入框

## 阶段 5：AI 检查功能

**目标**：AI 检查并展示三层结果。

- [ ] 创建 `src/lib/ai.ts`（AI 调用封装，支持多提供商）
- [ ] 设计三层检查 Prompt：
  - 语法纠错 Prompt
  - 表达优化 Prompt
  - 雅思评分 Prompt
- [ ] 创建 API 路由 `/api/check`
- [ ] 创建 `/check/[id]` 结果展示页面
  - 折叠面板（语法纠错 / 表达优化 / 整体评分）
  - 错误类型彩色标签
  - "加入笔记"按钮
- [ ] 处理 AI 调用失败的情况（提示用户检查 API Key）

**验证**：提交文章 → AI 检查 → 看到三层结果 → 每条错误都可点击"加入笔记"。
